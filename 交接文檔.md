## 專案目標 (Project Goal)

做一個泛用性網頁工具，把不同公司/品牌的格狀月排班表 Excel（欄位配置不固定）轉換成每日一分頁、橫向長條圖顯示的排班表 Excel，解決格狀班表不利於每日/即時人力控管的問題。

## 關鍵設定與上下文 (Key Context & Rules)

- 既有手動版 Excel/ODS 轉換邏輯：
  - 「整月外場班表」：格狀原始輸入，員工列 × 日期欄，每人占 2~3 列（可填分班上下班時間）
  - 每日分頁（"1"~"31"）：用 INDIRECT 依 AA 欄列號指標，從整月班表撈出當天上/下班時間，餵給橫向長條圖，時間軸範圍約 8-23 點
  - 既有檔案偶有列間距公式誤引前幾列的既有錯誤——這是既有檔案的問題，非本專案要修的對象，寫解析邏輯時不要照抄這類錯誤模式
- 圖表必須是 Excel/ODF 原生 chart 物件，不能用格子上色模擬
- **ODS格式技術踩坑筆記（供未來若需要用程式操作 ODS 檔案參考）**：
  - `chart:categories` 必須巢狀在 `<chart:axis chart:dimension="x">` 元素內部，直接用 `table:cell-range-address` 屬性即可，不需要 `<chart:domain>` 子元素包裝（已用 OASIS ODF 1.3 官方 RELAX NG schema 驗證：`chart-axis` 的內容模型是 `title? categories? grid*`）
  - ODF 圖表除了即時參照儲存格的定義外，還內建一份「local-table」快取快照（`<table:table table:name="local-table">`），是實際渲染畫面的依據；修改儲存格公式或强制重新計算（含 OpenOffice 的「強制重新計算」）都不會同步更新這份快取，需要直接改寫其 XML 內容
  - Python 沒有成熟的 ODS 寫入函式庫（openpyxl 不支援 ods），本次是直接讀寫 zip 內的 `content.xml` / `META-INF/manifest.xml` / `Object N/`（圖表物件）子文件；`mimetype` 檔案必須是 zip 內第一個項目且不可壓縮（ZIP_STORED），其餘可用 ZIP_DEFLATED
  - 在 Windows 上對已被 LibreOffice/OpenOffice 開啟的 ods 檔案寫入會遇到檔案鎖定（PermissionError / Device or resource busy），建議寫到 `.tmp` 暫存檔，等使用者確認關閉檔案後再 `mv` 取代正式檔名
- 泛用性辨識機制（2026-07-04 架構討論決策）：
  - 主要特徵（員工姓名、日期、員工編號、職稱）欄位位置不固定，機制為「啟發式猜測（關鍵字比對+型態嗅探）→ 使用者確認畫面（下拉選單修正）」，不追求全自動辨識
  - 次要特徵（例行行銷、節慶行銷、單店活動等訊息）為掛在特定日期上方的自由文字附註列，使用者於確認畫面圈出這些列即可，原樣依日期帶到輸出的每日班表，**不需要分類標籤**
  - 欄位對照確認後存成「範本」，新增 JSON 儲存層存欄位對照表；範本歸屬（同一來源判斷）採**使用者上傳時手動選擇範本**，不做自動指紋比對配對
  - 這代表工具不再是無狀態轉換器，需要新增持久化儲存元件
