## 專案目標 (Project Goal)

做一個泛用性網頁工具，把不同公司/品牌的格狀月排班表 Excel/ODS（欄位配置不固定）轉換成每日一分頁、橫向長條圖顯示的排班表 Excel 檔案，解決格狀班表不利於每日/即時人力控管的問題。**本工具是更大餐飲管理平台的子模組**，需具備多品牌/多店家泛用性，但不自建帳號/身分驗證系統。**架構決策：維持獨立微服務（FastAPI），與其他子專案透過 HTTP API 串接，不預期併入其他子專案的 codebase**（待平台主體技術棧確定後再視需求重新評估）。**輸出格式決策：僅提供 Excel（.xlsx）輸出，不提供 ODS 輸出**；上傳輸入端仍支援 .xlsx / .ods。

### 圖表標題加上月份（本輪，使用者需求）

- **需求**：圖表標題除了「N 日」之外要加上月份（如「3月1日 排班長條圖」），月份需從上傳的班表檔案內容辨識（不能寫死），且要同步套用於網頁預覽與輸出檔案圖表。
- **關鍵架構決策（已與使用者確認）**：月份**不寫入範本 JSON**，每次上傳都重新從當次檔案內容判斷。原因：範本設計上是跨月重複使用的（同一品牌每月上傳新檔案套用同一份已存範本），若把月份寫死進範本，下個月套用同一份範本時月份會卡在建立範本當下的舊值。
- **實作**：
  - `backend/heuristics.py` 新增 `guess_month(rows, header_row_idx, first_day_col, cols_per_day, max_days=31)`：掃描日期列在已知欄位間距下的儲存格，支援兩種來源——真正的 `datetime.date`/`datetime.datetime` 物件（直接取 `.month`）、或「N月M日」文字格式（regex 取出月份數字）。純數字日期欄位（如只有 1、2、3...沒有月份資訊）偵測不到月份時回傳 `None`。多數決（`Counter`，需 ≥50% 一致）避免個別雜訊值誤判。
  - `backend/parser.py` 的 `parse_schedule()`：讀完 rows、知道 `header_row`/`first_day_col`/`cols_per_day` 後呼叫 `guess_month()`，回傳值新增進結果字典的 `"month"` 欄位。
  - `backend/main.py`：`/api/preview` 回應新增 `"month"` 欄位；`/api/convert` 呼叫 `generate_xlsx(..., month=result["month"])`。
  - `backend/generator.py`：`generate_xlsx()`、`add_xlsx_gantt_chart()` 新增 `month=None` 參數，標題組法為 `f"{month}月{ws.title}日 排班長條圖"`（`month` 為 `None` 時退回原本「N 日」格式，不顯示月份，向下相容）。
  - `frontend/app.js`：新增模組層狀態 `currentMonth`，在 `confirmPreviewBtn` 的 `/api/preview` 回呼中賦值；`renderDayGantt()` 的 `dayTitle` 文字改為 `` currentMonth ? `${currentMonth} 月 ${day} 日 排班預覽` : `${day} 日 排班預覽` ``。
- **驗證**：用 10 份範例／真實客戶檔案全數重跑 `guess_month()`：7 月（白霧.ods、範例班表1MORE）、1 月（範例班表4TAO/5WANG、旺鍋、桃城雞排）、3 月（MR08）皆正確判斷；純數字日期欄位（範例班表2MOU/3MEMBER、MOU班表模組）正確回傳 `None`。`tests/verify_engine.py` 全數 `healthy=True` 無回歸。重啟 server 後直接打 `/api/convert` API 解壓確認 MR08 檔案標題正確顯示「3月1日 排班長條圖」。**使用者已透過瀏覽器實測確認成功**。

## 關鍵設定與上下文 (Key Context & Rules)

- **技術定位**：欄位辨識全部為規則式手寫邏輯，不是機器學習模型。完整規則表見 `README.md`。
- **架構決策：FastAPI + 獨立微服務**（2026-07-15 確認）。**輸出格式決策：僅 Excel，不提供 ODS**（2026-07-21 確認，上傳輸入不受影響）。
- **範本 JSON schema 的 `display` 區塊**：`{"axis_start": int, "axis_end": int}`（整點小時，0~24）。讀取點：`frontend/app.js` 的 `renderDayGantt()`、`backend/generator.py` 的 `generate_xlsx()`、`backend/heuristics.py` 的 `guess_axis_range()`；找不到時一律 fallback 8/24。
- **月份不是範本欄位**（本輪確認的重要設計決策）：`month` 是 `parse_schedule()` 每次呼叫時當場算出來的回傳值，**不會**寫進範本 JSON、也不會透過 `/api/templates` 存起來。原因是範本本身設計為跨月重複使用（同一品牌每月上傳新檔案套用同一份已存範本），若把月份寫死進範本會導致下個月套用時月份卡在舊值。frontend 對應用一個獨立的模組層狀態 `currentMonth`（非 `confirmedTemplate` 的一部分），在每次 `/api/preview` 回呼時重新賦值。**未來若有類似「這次分析出的資訊」需求，優先考慮走這個「回應層欄位」模式，而非塞進範本結構。**
- **openpyxl BarChart 的軸對應關係（務必記住）**：`chart.x_axis` 不論 `chart.type` 是 `"bar"`（橫向）或 `"col"`（縱向），永遠序列化成 `<c:catAx>`（類別軸，本專案對應姓名）；`chart.y_axis` 永遠是 `<c:valAx>`（數值軸，本專案對應時間）。這跟畫面上長條實際橫向或縱向排列**無關**。
- **openpyxl `chart.set_categories()` 呼叫時機**：必須在所有 `chart.add_data()` 呼叫**之後**才呼叫，否則分類資料不會出現在輸出檔案裡。
- **openpyxl 隱藏欄位＋圖表連動**：`ws.column_dimensions[letter].hidden = True` 隱藏欄位後，若該欄位是圖表資料來源，**必須**同時設定 `chart.visible_cells_only = False`，否則圖表預設只畫可見儲存格、資料會憑空消失。
- **openpyxl 類別軸標籤間隔**：`chart.x_axis.tickLblSkip`／`tickMarkSkip` 設為 `1` 理論上強制全部顯示，但經 2026-07-22（上一輪）本機重現測試證實，LibreOffice 並不會真正遵守這個屬性，仍會依圖表實際高度自行算間隔。此設定予以保留（無害）但不能單獨依賴它，真正解法是拉高 `chart.height`（見下一條）。
- **openpyxl 圖表高度**：`BarChart()` 預設 `chart.height = 7.5`（cm）、`chart.width = 15`（cm），員工數（類別數）再多也不會自動變高。`add_xlsx_gantt_chart()` 內 `chart.height = max(10, n_employees) * 0.8`，不設上限、下限比照 10 人的高度；寬度不需調整。**這行必須是函式內最後一次對 `chart.height` 賦值**，先前曾發生過重複賦值覆蓋掉修復的 bug（已排除），未來若要調整圖表尺寸，先搜尋整個函式確認沒有重複賦值。
- **openpyxl 移除圖例**：`chart.legend = None`（預設是一個 `Legend()` 物件，會顯示在圖表右側）。
- **【本輪重要教訓】調整既有數值軸的 `crosses`/`axPos`/`tickLblPos` 屬性來改變軸線顯示位置（例如想把 bar 型圖表的時間軸從下方搬到上方），在 LibreOffice 與 Google 試算表上都被證實無效（屬性正確寫入 XML，但渲染結果完全不受影響）。**這不是單一軟體的 bug，是這類手法本身在多數試算表軟體實作裡的支援度就很低**。若未來還有「把某條軸換到另一側/另一端」的需求，**直接採用次座標軸技術**（見下一條），不要再花時間嘗試調整既有單軸的這幾個屬性。
- **openpyxl 次座標軸（雙軸圖表）技術**：`chart1 += chart2` 可以合併兩個圖表物件的繪圖區。若 `chart2` 的某個軸（如 `chart2.y_axis.axId`）設成與 `chart1` 不同的數值，該軸會成為次座標軸；預設情況下次座標軸會渲染在與主座標軸相對的另一側（例如主座標軸在下方，`crosses="max"` 的次座標軸會在上方）。若只需要次座標軸本身、不需要 `chart2` 自己的類別軸重複畫出來，設定 `chart2.x_axis.delete = True`。`chart2` 的資料 series 若不想被實際看到，設 `graphicalProperties.noFill = True`。完整範例見 `backend/generator.py` 的 `add_xlsx_gantt_chart()` 內 `top_axis_chart` 那段程式碼與其註解。

- **`/api/preview` 回傳完整 `employees` 陣列與 `month`**，供前端單日預覽自由切換任一天並顯示正確標題。
