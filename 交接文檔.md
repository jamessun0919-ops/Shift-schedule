## 專案目標 (Project Goal)

做一個泛用性網頁工具，把不同公司/品牌的格狀月排班表 Excel/ODS（欄位配置不固定）轉換成每日一分頁、橫向長條圖顯示的排班表 Excel 檔案，解決格狀班表不利於每日/即時人力控管的問題。**本工具是更大餐飲管理平台的子模組**，需具備多品牌/多店家泛用性，但不自建帳號/身分驗證系統。**架構決策：維持獨立微服務（FastAPI），與其他子專案透過 HTTP API 串接，不預期併入其他子專案的 codebase**（待平台主體技術棧確定後再視需求重新評估）。**輸出格式決策：僅提供 Excel（.xlsx）輸出，不提供 ODS 輸出**；上傳輸入端仍支援 .xlsx / .ods。

- **範本 JSON schema 的 `display` 區塊**：`{"axis_start": int, "axis_end": int}`（整點小時，0~24）。讀取點：`frontend/app.js` 的 `renderDayGantt()`、`backend/generator.py` 的 `generate_xlsx()`、`backend/heuristics.py` 的 `guess_axis_range()`；找不到時一律 fallback 8/24。
- **月份不是範本欄位**：`month` 是 `parse_schedule()` 每次呼叫時當場算出來的回傳值，不會寫進範本 JSON。frontend 對應用獨立的模組層狀態 `currentMonth`。
- **每小時上班人數不是範本欄位，也不快取**：`compute_hourly_headcount()`（後端）與 `computeHourlyHeadcount()`（前端 JS，本輪新增）皆是每次呼叫時當場算出，**兩邊邏輯必須保持一致**（重疊時數比例加總，非二元判斷）。若之後任一邊的計算規則調整，務必同步修改另一邊，否則 xlsx 下載檔與網頁預覽會出現數字不一致。
- **【本輪重要】每小時人數是「加總後的比例」，不是「有無半點交界」的旗標**：偶數位員工同時有半小時重疊時，加總可能剛好變回整數（如 6×0.5=3.0），這是正確結果，不代表該小時沒套用小數計算。之後若使用者或自己覺得「這格看起來沒有修正生效」，先手算逐位員工的重疊時數確認，不要預設是程式漏算。
- **甘特圖姓名軸新增「上班人數」虛擬列**：工作表第 2 列固定保留給這個虛擬類別，真實員工資料從第 3 列開始（`r = idx + 3`，非原本的 `idx + 2`）。之後若要再調整這個區塊的程式碼，務必注意所有「列位置」相關計算（`last_row`、`chart.height` 的類別數、`cats` Reference 範圍）都已經把這一列算進去。
- **openpyxl BarChart 的軸對應關係（務必記住）**：`chart.x_axis` 不論 `chart.type` 是 `"bar"`（橫向）或 `"col"`（縱向），永遠序列化成 `<c:catAx>`（類別軸，本專案對應姓名）；`chart.y_axis` 永遠是 `<c:valAx>`（數值軸，本專案對應時間）。這跟畫面上長條實際橫向或縱向排列**無關**。
- **openpyxl `chart.set_categories()` 呼叫時機**：必須在所有 `chart.add_data()` 呼叫**之後**才呼叫，否則分類資料不會出現在輸出檔案裡。
- **openpyxl 隱藏欄位＋圖表連動**：`ws.column_dimensions[letter].hidden = True` 隱藏欄位後，若該欄位是圖表資料來源，**必須**同時設定 `chart.visible_cells_only = False`，否則圖表預設只畫可見儲存格、資料會憑空消失。
- **openpyxl 圖表高度**：`add_xlsx_gantt_chart()` 內 `chart.height = max(10, n_employees + 1) * 0.8`。**這行必須是函式內最後一次對 `chart.height` 賦值**，先前曾發生過重複賦值覆蓋掉修復的 bug（已排除），未來若要調整圖表尺寸，先搜尋整個函式確認沒有重複賦值。
- **欄位英文字母換算務必用 `_col_letter()`，不要用 `chr(65+n)`**：輔助欄總數容易超過 26 欄。
- **openpyxl 的 `DataLabel.numFmt` 只能設定 `formatCode`，沒有介面可設定 OOXML 規格要求的 `sourceLinked` 屬性**：需在 `wb.save()` 之後呼叫 `_fix_datalabel_numfmt_sourcelinked(output_path)`（存檔後 XML 後處理，正規表示式補上 `sourceLinked="0"`）。
- **自訂 `DataLabel` 務必明確關閉 `showCatName`/`showSerName`（連同 `dLbls` 這一層一起關）**：完整範例見 `add_xlsx_gantt_chart()` 內 `dlbls_common`。
- **openpyxl 類別軸標籤間隔**：`tickLblSkip`／`tickMarkSkip` 設為 `1` 對 LibreOffice 無實際效果，真正解法是拉高 `chart.height`。
